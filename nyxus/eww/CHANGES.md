## Honesty / tradeoffs (eww is GTK3 CSS, 2D)

- **No live 3D / particle physics.** The comet is a moving CSS gradient (bright head +
  tapered tail), not a particle system. A photoreal fiery comet would require a
  pre-rendered looping WebP/APNG sprite (drop-in point: `assets/`, referenced via a
  moving `background-position` layer) â€” not yet wired.
- **No `clip-path` in GTK3** â€” non-rectangular "shard" shapes are not possible; bars use
  rounded rectangles.
- **Graffiti "reveal under comet"** is a visual approximation (faint texture + bright
  comet pass), not a true masked spotlight â€” GTK CSS has no moving mask.

  animated pinpoint stars + 16-frame twinkle + faint nebula haze) -
  regenerated, not layered over.

  core, tail amber `rgba(255,200,120)` -> violet `rgba(178,128,255)` ->
  magenta `rgba(255,60,172)` -> transparent. Ember gold stays as the

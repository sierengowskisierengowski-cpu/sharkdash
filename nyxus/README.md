# NYXUS · The Real Build

This is a permanent GitHub backup of **your actual daily-driver Hyprland
build** — the elaborate "Signature Edition" desktop (prism gradients,
frosted glass, eww flyouts, hyprlock Prism HUD, security center, living
wallpaper, etc.) — as opposed to [`../nexus/`](../nexus/), which is a
deliberately minimal, dependency-free **safe-mode recovery config** kept
around specifically so a broken machine can always get back to a working
desktop from a USB stick or another PC.

**Do not merge these two.** `nexus/` must stay minimal so it keeps working
as a rescue kit even when NYXUS's daemons/scripts/services are missing or
broken. `nyxus/` is allowed to be as elaborate and dependency-heavy as you
want, because it's just a backup — if it doesn't load, you fall back to
`nexus/`.

## Recovery history (2026-07-11)

1. Config was migrated to `hyprland.lua` — this wiped almost all
   customization (the generated Lua file was the untouched autogenerated
   template, only `fileManager`/`menu` were customized).
2. A touchpad gesture (`gesture = 3, horizontal, workspace`) combined with
   an aquamarine/libinput bug caused a crash-to-login-screen loop on
   opening any app.
3. A cloud agent broke the loop by switching to the minimal `nexus/`
   `.conf` build (stable, but not your real desktop).
4. This `nyxus/` folder was rebuilt from the pre-Lua real build
   (`~/.config/hypr.broken/` at the time of recovery) with two fixes:
   - The touchpad workspace-swipe gesture line was removed entirely.
     `gestures:workspace_swipe` was removed from Hyprland in 0.51+ — the
     *only* way to disable the swipe now is to not define a
     `gesture = ..., workspace` line at all. (Confirmed via
     `Hyprland --verify-config`.)
   - `$fileManager = dolphin` / `$menu = hyprlauncher` set explicitly, with
     safe `sh -c` fallbacks to installed alternatives (neither binary was
     installed on the recovery machine).
   - `hypridle.conf`'s lock listener calls `hyprlock` directly instead of
     `loginctl lock-session` (guarantees the lock actually fires).

Verified with `Hyprland --verify-config` (Hyprland 0.55.4): **config ok**,
zero errors — before this recovery, even the "stable" `nexus/` config had
3 silently-ignored config errors (deprecated `dwindle:pseudotile`,
`misc:vfr`, `gestures:workspace_swipe`, plus an invalid `togglesplit`
dispatcher). Those don't crash Hyprland (they just print in the on-screen
`configerrors` overlay), but they're cleaned up here anyway.

## What's NOT in this repo

Wallpapers/videos (`walls/`, ~168MB of PNGs + MP4s) are intentionally
**excluded** to keep this repo light. They're preserved locally at:

- `~/.config/hypr/walls/` (live)
- `~/nyxus-build-recovery/` (full backup made during recovery, includes a
  single `nyxus-hypr-full-backup-*.tar.gz` with everything)

If you want the wallpapers version-controlled too, use `git lfs track
"nyxus/hypr/walls/**"` before adding them — don't commit them raw.

Also not included: the `~/.nyxus/*.py` apps, `~/.local/bin/nyxus-*` and
`/usr/local/bin/nyxus-*` scripts, and `nyxus-*.service` systemd units that
the config's `exec-once`/`bind` lines call out to. Those live outside
`~/.config/hypr` and aren't part of "the Hyprland config" — back those up
separately if you want the same insurance for them.

## Restore

```bash
cp -r nyxus/hypr/. ~/.config/hypr/
# wallpapers aren't in git — restore them from ~/nyxus-build-recovery/ or
# your local ~/.config/hypr/walls/ if that machine still has them.
hyprctl reload
```
